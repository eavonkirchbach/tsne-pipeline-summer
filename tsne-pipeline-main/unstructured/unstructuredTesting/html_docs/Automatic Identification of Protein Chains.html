Automatic Identification of Protein Chains

Several wizards attempt to identify protein chains based on annotations within the file when importing data. For example, when importing a vector in Genbank .gb format that is supposed to code for a mature heavy chain of an antibody, the respective sequence is looked up by scanning feature annotations found in the file.

The identification strings are interpreted as Java regular expressions. The matching is case insensitive if the pattern is prefixed with the expression (?i).

Genbank .gb Import

These values are used when importing from a Genbank .gb-formatted file. For example, if the protein is a Two Chain Antibody (e.g. Fab or IgG), accepted feature annotations in the file either read mature light chain or mature LC in order to identify the nucleotides coding for the Light Chain of the protein. The feature annotations need to be present either in the label or the note qualifier, e.g. /label="mature LC". For GenPept files (Genbank-formatted file containing peptides) the feature annotations are searched for in the same way.
